React 简介

  • 库：小而巧，优点可以很方便的从一个库，切换到另外一个库，代码及会不会改变。只提供了特定的API。
  • 框架：大而全，框架提供了一整套的解决方案。所以在项目中间切换到另外框架，是比较困难的。

前端三大主流框架

  • Angular.js,最早
  • Vue.js,最火，关注的人比较多
  • React.js,最流行，用的人比较多

React 和 Vue 的对比，组件化方面：

  • 模块化：是从代码角度来进行分析，把一些可复用的代码抽离为单个的模块，便于项目的维护和开发。
  • 组件化：是从UI界面角度进行分析，把一些可复用的UI元素抽离为单独的组件，便于项目的维护和开发。
  • 组件化的好处：随着项目规模的增大，手里的组件越来越多，很方便就能把现有的组件拼接出来。
  • Vue中实现组件：通过.vue文件，来创建对应的组件，由template（结构）,script（行为）,style（样式）三部分组成。
  • Recat中实现组件：Recat中有组件化的概念，但是并没有像vue这样的组件模板文件，React中，一切都是以JS来表现得。因此JS要合格，ES6和ES7要会用。

React 和 Vue 的对比，开发团队方面

  • React是由FaceBook前端团队进行维护和更新的，因此，React的维护开发团队，技术实力雄厚。
  • Vue，第一版主要是由作者尤雨溪专门进行维护的，Vue2，由以他为主的一个小团队来进行相关的开发和维护。

React 和 Vue 的对比，社区方面

  • 社区强大，一些常见的问题，坑，最优的解决方案，文档，博客在社区中都是可以方便找到的。
  • 它的社区小一点，有一些坑，可能没人踩过。

React 和 Vue 的对比，移动APP开发体验方面

  • React，结合ReactNative这门技术
  • Vue，结合Weex这门技术,最流行，有成功案例。
